# Software Proverbs

Short sayings about software, memory, and tools.

- Data without backup is a rumor.
- Logs are the memory of systems.
- A good name is half a design.
- The simplest tool is often the one that survives.
- Public does not mean important, but it does mean discoverable.
- Private does not mean worthless.
- A commit is a small promise to the future.
- A README is a handshake with a stranger.
- Every abstraction eventually asks to be understood.
- The bug you cannot reproduce still happened to someone.
- A temporary script may outlive the system it was meant to fix.
- A cache is a place where the past pretends to be fast.
- A migration is a bet that the future will understand the past.
- A backup is only real after it has been restored.
- Naming is where design becomes language.
- An error message is part of the user interface.
- Good tools reduce drama.
- Quiet software respects attention.
- A deleted file may survive in more places than expected.
- The cloud is not weightless; it is merely elsewhere.
- Documentation is memory with formatting.
- A version number is a small calendar.
- The most durable format is often plain text.
- A link is a promise the web may not keep.
- The future prefers things that can still be opened.

## 한국어로

- 백업 없는 데이터는 소문이다.
- 로그는 시스템의 기억이다.
- 좋은 이름은 설계의 절반이다.
- 가장 단순한 도구가 가장 오래 살아남는 경우가 많다.
- 공개되었다는 것은 중요하다는 뜻이 아니라, 발견될 수 있다는 뜻이다.
- 비공개라는 것은 가치 없다는 뜻이 아니다.
- 커밋은 미래에게 보내는 작은 약속이다.
- README는 낯선 사람에게 건네는 첫 악수다.
- 문서는 서식이 있는 기억이다.
- 가장 오래 살아남는 형식은 종종 평범한 텍스트다.
