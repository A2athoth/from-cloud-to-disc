# Exhibits

Small museum-like readings of repository objects. These exhibits are kept together so they read as a single compact gallery rather than scattered fragments.

## Exhibit 001: The Repository

A repository is usually treated as a working place.

Here it is treated as a display case.

The files are not steps toward a program. They are the artifact itself.

## Label

Public Git repository, early twenty-first century.

Used here as an archive, a timestamp, and a message container.

## Exhibit 002: The Commit

A commit is a small statement that change happened.

It does not preserve intention perfectly.

It preserves enough to say: before this, the artifact was different.

## Label

Versioned memory unit.

Commonly used in software development, repurposed here as a record of editorial decisions.

## Exhibit 003: The Disc

The compact disc is not modern in the way the cloud is modern.

That is why it matters here.

It gives a digital record edges, weight, and inconvenience.

## Label

Optical storage medium.

Obsolete enough to be poetic; functional enough to remain plausible.

## Exhibit 004: The README

A README is usually the first handshake between a project and a stranger.

In this archive, the README is also a threshold.

It explains that nothing needs to run for the artifact to work.

## Label

Entrance text.

A document that turns a folder into a place someone can enter.

## Exhibit 005: The Checksum

A checksum does not explain a file.

It only notices whether the bytes changed.

That small power is useful in an archive because meaning often depends on fragile sameness.

## Label

Integrity marker.

A number that cannot preserve meaning, but can detect disturbance.
