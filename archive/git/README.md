# Git as Archive Layer

This page gathers the small Git-native notes that were previously split across several short files.

## History as Artifact

The files are the visible surface.

The history is another layer.

In a normal software project, history often explains how a product was built.

Here, history explains how the archive decided what it wanted to be.

## Commit as Memory

A commit is not the past.

It is a compact claim about the past.

It says that a set of files once looked this way, under this message, at this point in the archive's history.

That is not perfect memory.

It is useful memory.

## Public Repository as Place

A public repository is not only a container.

It is a place with an address, an entrance, a visible history, and an implied audience.

Even if no one visits, it remains available.

That availability is part of the artifact.

## Branch as Possible Future

A branch is a future that has not yet been accepted by the main story.

Most branches disappear.

Some merge.

Some remain as evidence that another direction was possible.

This archive mostly moves on `main`, but the idea of branching still matters: every revision is a choice among possible futures.
