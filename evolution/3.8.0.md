# Evolution Note: 3.8.0

## Repository state reviewed

At 3.7.0, folder README files covered their sibling files, but many generated contents lines repeated obvious format labels such as `(Markdown document)`.

## Reconsideration

A local map should help readers understand purpose, not restate file extensions. If a file already ends in `.md`, saying that it is a Markdown document adds noise.

## Changes made

- Removed repetitive format labels from README contents sections.
- Kept sibling-file coverage intact.
- Preserved or added useful descriptions where filenames alone would be too opaque.
- Updated metadata, inventory, and checksums.

## Result

The README contents sections remain complete, but they read less mechanically.
