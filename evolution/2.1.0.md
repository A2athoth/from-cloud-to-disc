# Evolution Note: 2.1.0

## Repository state reviewed

At 2.0.0, the archive was rich but the top-level documents overlapped too much.

## Reconsideration

Redundancy is useful in an archive, but entry points should have distinct jobs. Reviewer guides also needed to exist before other entry points could link to them.

## Changes made

- Shortened and clarified the README files.
- Reassigned roles for README, INDEX, ARCHIVE_MAP, COLLECTION, and READING_PATHS.
- Added English and Korean reviewer guides as stable entry points.
- Reduced repeated long lists in the top-level documents.

## Result

The archive became easier to enter without removing its intentional redundancy.
