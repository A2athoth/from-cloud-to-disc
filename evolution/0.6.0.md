# Evolution Note: 0.6.0

## Repository state reviewed

At 0.5.0, the archive had strong human-facing structure and a preservation protocol, but machine-readable identity remained thin.

## Reconsideration

If the repository claims semantic interest, it should contain semantic and catalog metadata directly.

## Changes made

- Added Dublin Core-style metadata.
- Added a Data Package descriptor.
- Added RDF/Turtle catalog metadata.
- Added a citation file.
- Documented the metadata folder.

## Result

The archive became more legible to machines while remaining readable to humans.
