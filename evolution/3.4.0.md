# Evolution Note: 3.4.0

## Repository state reviewed

At 3.3.0, the archive had a cleaner root and an explicit bilingual-balance policy, but several references still described the pre-cleanup tree.

The issue was not a broken concept. It was consistency: the human-facing guide layer, the machine-readable map, and some preservation notes no longer described the same structure.

## Reconsideration

A public artifact should be modest, but its modesty should not look unfinished.

If the repository asks future readers to trust its inventory, checksums, and preservation notes, then its path references and version metadata need to be as careful as its prose.

## Changes made

- Corrected stale path references in context, accessibility, preservation, print, and data files.
- Updated the root-structure and 3.3.0 evolution notes to distinguish removed administrative files from kept entry points.
- Added clearer navigation for folder layers and guide documents.
- Added Korean bilingual-balance guidance and a Korean essay about having no project to show.
- Replaced the short CC0 summary with the standard CC0 1.0 legal code.
- Expanded `CITATION.cff` and updated machine-readable version fields to `3.4.0`.
- Regenerated the file inventory and checksum list.

## Result

The archive remains intentionally small in tone, but its current structure, metadata, and public surface now agree with each other more closely.
