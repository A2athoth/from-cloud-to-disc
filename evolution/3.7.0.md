# Evolution Note: 3.7.0

## Repository state reviewed

At 3.6.0, the root and internal folder structure had been compressed, but some folder README files did not mention every file in the same folder.

## Reconsideration

A compressed archive still needs local discoverability. If a reader opens a folder README, it should not silently omit sibling files that are part of that folder.

## Changes made

- Audited every README files file against its sibling files and subfolders.
- Added explicit contents sections where files or subfolders were not locally referenced.
- Kept the sections concise by using each file's heading or file role as the description.
- Updated metadata, inventory, and checksums.

## Result

Folder README files now act as local maps as well as prose introductions, reducing the chance that a file is physically present but logically invisible.
