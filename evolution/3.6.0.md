# Evolution Note: 3.6.0

## Repository state reviewed

At 3.5.0, the root directory had been compressed, but several internal folders still contained many short fragments.

## Reconsideration

A smaller root is not enough if the next layer still feels scattered. The internal folders should also prefer compact gathered documents when individual files are only brief notes.

## Changes made

- Collapsed future-facing notes into `archive/future/README.md`.
- Collapsed curation rules into `guide/curation/README.md`.
- Collapsed Korean archive notes into `ko/archive/README.md`.
- Merged short context background notes into `context/background.md`.
- Merged language structure notes into `language/README.md`, while keeping glossary and untranslatable-word files separate.
- Updated path references, metadata, inventory, and checksums.

## Result

The archive keeps the same conceptual layers but has fewer internal fragments. Short text now more often appears as a section inside a stronger document rather than as a standalone file.
