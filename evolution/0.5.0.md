# Evolution Note: 0.5.0

## Repository state reviewed

At 0.4.0, the repository had a stronger physical-disc identity, but it still lacked an explicit preservation protocol.

## Reconsideration

A time capsule should not merely contain reflections about preservation. It should also state how it tries to preserve itself.

## Changes made

- Added a preservation folder.
- Documented format preferences, verification, migration, redundancy, and bit rot.
- Reaffirmed plain text as the primary preservation strategy.

## Result

The repository became more like an intentional archive and less like a collection of adjacent essays.
