# Evolution Note: 3.3.0

## Repository state reviewed

At 3.2.0, the archive was submission-ready, but the root directory had become too crowded and some files were too administrative for the final source tree.

## Reconsideration

A public artifact should not expose too much of its submission workflow as source content. The root should act as an entrance, not as a filing cabinet.

The bilingual structure also needed an explicit policy: balance should be based on reader experience, not strict one-to-one file count.

## Changes made

- Kept short descriptions and reviewer guides as reader entry points.
- Removed GitHub About recommendation files because the repository description and topics had already been set.
- Reframed submission-oriented materials as `context/` documents.
- Moved guide documents into `guide/`.
- Moved core archive texts into `core/`.
- Added root-structure and bilingual-balance review documents.

## Result

The repository became less administrative, less cluttered at the root, and clearer about why Korean does not need mechanical one-to-one parity with English.
