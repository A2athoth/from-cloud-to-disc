# File Format Policy

Preferred formats:

- Markdown for human-readable structured text
- plain text for labels and short notes
- JSON / JSON-LD for machine-readable metadata
- YAML only when convention makes it useful
- Turtle for lightweight semantic cataloging

Avoided formats:

- binary-only documents
- compressed bundles as primary records
- images required for understanding
- executable files required for reading

A reader should be able to understand the archive with a text editor.
