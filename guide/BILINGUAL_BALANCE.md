# Bilingual Balance Policy

This repository is bilingual, but it is not a strict line-by-line translation project.

## Principle

English is used for reach.  
Korean is kept for origin.

That means the English and Korean layers should be balanced by role, not by identical file count.

## What should match one-to-one

High-level entry points should have Korean counterparts:

- `README.md` / `README.ko.md`
- `INDEX.md` / `INDEX.ko.md`
- `guide/ARCHIVE_MAP.md` / `guide/ARCHIVE_MAP.ko.md`
- `guide/REVIEWER_GUIDE.md` / `guide/REVIEWER_GUIDE.ko.md`
- `context/short-description.md` / `context/short-description.ko.md`

These files help readers enter the archive.

## What should not match one-to-one

Interpretive and archival files do not need strict parity.

Some English files are broad navigation or metadata support. Some Korean files are native reflections that should not be forced into English-shaped mirrors.

A perfect file-count match would make the archive larger but not necessarily clearer.

## Target balance

- Entry paths: paired.
- Core identity: paired or clearly reachable from both languages.
- Korean origin: allowed to be independent.
- Metadata and checksums: shared.
- Deep archive: curated, not mirrored mechanically.

The goal is not symmetry.

The goal is for both English-first and Korean-first readers to understand what the artifact is and why it exists.
