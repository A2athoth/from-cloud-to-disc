# FAQ

## Is this a software project?

No. It uses a software repository as a public archival form.

## Why GitHub?

Because GitHub brings code, history, publicness, and preservation into one visible place.

## Why plain text?

Plain text is easy to copy, inspect, diff, quote, print, and archive.

## Why bilingual?

English helps the artifact travel. Korean preserves the language in which the idea began.

## Is it finished?

It is complete enough to explain itself. Future changes should improve clarity rather than size.
