# Curation

This page gathers the archive growth and selection rules. The goal is to keep the repository meaningful without letting it become a filing cabinet.

## Curation Principles

## 1. Prefer clear entrances

A new document should help a reader enter the archive or understand its structure.

## 2. Prefer durable questions

Statements may age quickly. Questions often remain useful longer.

## 3. Prefer plain text

If a document cannot be read in a basic text editor, it needs a strong reason to exist here.

## 4. Preserve origin and context

The archive should preserve not only contents, but the conditions that produced them.

## 5. Prefer enough

The archive should be complete enough to stand, but not so large that its shape disappears.

## Selection Criteria

A proposed addition should answer at least two questions:

1. Does it strengthen the cloud-to-disc concept?
2. Does it help a future reader understand the artifact?
3. Does it improve the Korean/English structure?
4. Does it make the archive easier to preserve, verify, or navigate?
5. Does it add a new angle rather than repeat an existing one?

If the answer is mostly no, the better choice is not to add it.

## What Not to Add

Avoid adding:

- files that only repeat the core statements
- screenshots required for understanding
- binary files that replace text
- generic filler essays
- jokes that weaken the quiet tone
- automation that becomes the center of attention
- personal details that do not serve the archive
- files added only to increase file count

A public archive can be small and still complete.
