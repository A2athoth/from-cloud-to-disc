# RFC 0001: Repositories as Artifacts

## Summary

A repository can be more than a container for code.

It can also be a public artifact, a timestamp, a small archive, or a message to an unknown future reader.

## Motivation

Many repositories exist to build or distribute software.

This one exists to preserve a thought in a format usually used for software.

That change of purpose is the point.

## Proposal

A repository may be considered an artifact when its structure, history, and availability are part of its meaning.

The files matter.

The commit matters.

The public location matters.

The possibility of becoming physical matters.

## Non-goals

This RFC does not claim that every repository is art.

It only claims that a repository can be designed as an artifact when the author intends it that way.
